# MTC 1ClickPlot Pro (Revit C# Add-in)

โปรเจกต์นี้เป็น Add-in สำหรับ Autodesk Revit ที่พัฒนาด้วยภาษา C# (.NET Framework) ทำหน้าที่จัดการการพิมพ์แบบ (Plot) ออกเป็น PDF อัตโนมัติ โดยมีจุดเด่นคือ:
1. สามารถเปลี่ยนสีเส้นทั้งหมด (Object Styles) เป็นสีดำชั่วคราวตอนสั่ง Print และคืนค่าสีเดิมเมื่อพิมพ์เสร็จ
2. ตรวจจับคิวพิมพ์จาก Windows Spooler เพื่อรอให้การพิมพ์เสร็จสมบูรณ์
3. ระบบ Auto-Rename เปลี่ยนชื่อไฟล์ตาม `Sheet Number` และย้ายไฟล์ไปยังโฟลเดอร์ที่ผู้ใช้เลือกโดยอัตโนมัติ

---

## 📂 โครงสร้างโฟลเดอร์ (Folder Structure)

โปรเจกต์นี้ใช้หลักการ **Separation of Concerns (การแบ่งแยกหน้าที่)** เพื่อให้โค้ดเป็นระเบียบและบำรุงรักษาง่าย โดยแบ่งออกเป็นส่วนต่างๆ ดังนี้:

### 1. ไฟล์หลัก (Root Files)
- **`App.cs`** 
  - **หน้าที่:** จัดการหน้าตาบน Revit (Ribbon UI)
  - **คำอธิบาย:** ทำหน้าที่สร้าง Tab, Panel และปุ่มคำสั่ง (Button) บนเมนูของ Revit และกำหนดให้ปุ่มนั้นเรียกใช้งานคำสั่งอะไร
- **`Command.cs`** 
  - **หน้าที่:** ศูนย์กลางควบคุมการทำงาน (Controller)
  - **คำอธิบาย:** เป็นจุดเริ่มต้นเมื่อผู้ใช้กดปุ่ม ทำหน้าที่เรียกหน้าต่าง UI ขึ้นมาแสดงผล และควบคุมลำดับขั้นตอน (Workflow) เช่น สั่งให้เปลี่ยนสี -> สั่ง Print -> เรียกคืนสี -> สั่งเปลี่ยนชื่อไฟล์
- **`Config.cs`**
  - **หน้าที่:** เก็บค่าคงที่ (Constants)
  - **คำอธิบาย:** ใช้เก็บตั้งค่าส่วนกลาง เช่น ชื่อ Printer เป้าหมาย, พาธโฟลเดอร์ Temp (เช่น `C:\Revit_PDF_Temp`), หรือหมวดหมู่ที่ต้องยกเว้น

### 2. โฟลเดอร์ `UI\` (User Interface)
- **`PrintUI.xaml`** 
  - **หน้าที่:** หน้าต่างกราฟิก (View)
  - **คำอธิบาย:** เขียนด้วยภาษา XAML ใช้จัดวางหน้าตาของหน้าต่าง เช่น ปุ่ม Checkbox, ช่องค้นหา, และปุ่ม Browse โฟลเดอร์
- **`PrintUI.xaml.cs`** 
  - **หน้าที่:** โค้ดเบื้องหลังหน้าต่าง (Code-behind)
  - **คำอธิบาย:** รับค่า Event ต่างๆ เช่น เมื่อผู้ใช้กดปุ่ม "RUN PROCESS" จะทำการเก็บข้อมูลว่าติ๊กเลือกอะไรบ้างส่งกลับไปให้ `Command.cs`

### 3. โฟลเดอร์ `Managers\` (Core Logic)
- **`RevitDataManager.cs`**
  - **หน้าที่:** จัดการข้อมูลที่เกี่ยวกับ Revit API ทั้งหมด
  - **คำอธิบาย:** รวบรวมฟังก์ชันสำหรับดึงรายชื่อ Sheet, ดึงรายการ Print Setting, ดึง Categories ทั้งหมด รวมถึงรับหน้าที่เปลี่ยนสีเส้น และส่งคำสั่ง Print ให้กับ Revit
- **`PrintSystemManager.cs`**
  - **หน้าที่:** จัดการระบบคอมพิวเตอร์และไฟล์ (Windows System)
  - **คำอธิบาย:** คุยกับ Windows นอก Revit เช่น การดึงชื่อ Printer ในเครื่อง, ระบบดักรอไฟล์ (Spooler), และระบบจับไฟล์มาเปลี่ยนชื่อ (Regex) แล้วย้ายไปยังเป้าหมาย

## 🛠️ วิธีการแก้ไขและอัปเดตโค้ด (How to Update)

หากต้องการแก้ไขคำสั่ง หรือเพิ่มฟีเจอร์ใหม่ ให้ดำเนินการดังนี้:
1. เปิดไฟล์ `MTC_1ClickPlot.csproj` ด้วย **Visual Studio**
2. ทำการแก้ไขไฟล์ `.cs` หรือ `.xaml` ตามต้องการ
3. คลิกขวาที่โปรเจกต์ เลือก **Build** (หรือกด `Ctrl + Shift + B`)
4. โปรแกรมจะทำการสร้างไฟล์ `MTC_1ClickPlot.dll` ตัวใหม่ขึ้นมา

---

## 🚀 วิธีการติดตั้งและแจกจ่าย (Deployment)

หลังจาก Build สำเร็จ การนำไปติดตั้งลงเครื่องผู้ใช้งาน มีขั้นตอนดังนี้:
1. นำไฟล์ **`MTC_1ClickPlot.dll`** (อยู่ที่โฟลเดอร์ `bin\Release\`) 
2. นำไฟล์ **`MTC_1ClickPlot.addin`** (อยู่ที่โฟลเดอร์หลัก)
3. นำทั้ง 2 ไฟล์ไปวางไว้ที่:
   👉 `%appdata%\Autodesk\Revit\Addins\2024`
4. เปิดโปรแกรม Revit 2024 โปรแกรมจะโหลด Add-in และสร้างปุ่มขึ้นมาพร้อมใช้งานทันที

*(หมายเหตุ: หากเป็นการอัปเดตโค้ดในภายหลัง ไม่ต้องอัปเดตไฟล์ `.addin` ให้นำเฉพาะไฟล์ `.dll` ไปวางทับไฟล์เดิมในขณะที่ปิดโปรแกรม Revit อยู่)*
